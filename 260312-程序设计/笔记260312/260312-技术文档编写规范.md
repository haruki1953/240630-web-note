# 📘 技术文档编写规范（可作为你自己的标准）

## 🎯 1. 文档的核心理念

### 🧩 **1.1 同构递归（Homomorphic Recursion）**

你的想法非常好：  
**每个层级都可以重复使用“需求 → 架构 → 实现”这一套结构。**

例如：

- 整个系统：需求 → 架构 → 实现
- 某个子系统（前端）：需求 → 架构 → 实现
- 某个模块（组件/函数）：需求 → 架构 → 实现

这让文档结构天然统一、可扩展、可深入。

### 🏗️ **1.2 异构分层（Heterogeneous Layering）**

不同层级的内容密度不同：

|层级|内容特点|示例|
|---|---|---|
|系统级|全局视角、跨模块关系、技术选型|整体架构图、数据流、技术栈|
|子系统级|模块职责、接口、内部结构|前端架构、后端架构、硬件架构|
|组件/函数级|具体实现细节|API 设计、状态机、算法流程|

这让文档既不臃肿，也不缺细节。
